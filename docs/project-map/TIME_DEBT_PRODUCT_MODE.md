# TIME_DEBT_PRODUCT_MODE｜Time Debt 产品模式地图

## 1. 新目标

Time Debt 不只是计时器，而是：

- 今日时间使用仪表盘
- 明日时间布局
- 实际执行反馈
- 浮窗快速记录
- 分类 / 任务复用

核心循环是：

```text
计划时间 -> 实际记录 -> 偏差反馈 -> 明日调整
```

第一阶段只记录产品地图，不直接开发完整首页仪表盘、未完成处理流或浮窗字段增强。

## 2. 两种使用模式

### 反映模式

- 不提前规划，只记录当天真实时间使用。
- 重点是自动统计、图表、分类占比和复盘线索。
- 用户不需要像机器人一样提前排满每天。

### 布局模式

- 提前一天安排第二天时间。
- 第二天按计划执行，并对比计划与实际偏差。
- 重点是时间块、实际记录、偏差反馈和明日调整。

共同原则：

- 不规划也能用。
- 规划了就能对比。
- 未完成不等于失败，应进入复盘、迁移或调整。

## 3. 首页仪表盘方向

未来 Time Debt 首页应从记录列表优先转向 Dashboard：

- 今日分类占比
- 今日总记录时长
- 当前正在进行
- 计划 vs 实际
- 明日布局入口
- 未完成处理入口
- AI 赋能占比概览

第一阶段优先级：

1. 今日分类占比
2. 今日总记录时长
3. 当前正在计时
4. 计划 vs 实际占位区

本轮不接入 ECharts，不实现完整仪表盘。

## 4. 未完成任务处理策略

未来未完成任务不显示为简单失败，而是支持：

- 迁移到明天
- 放弃
- 拆小
- 标记为计划过载
- 写入复盘

入口可以先出现在首页仪表盘中，完整处理流后续单独开发。

## 5. 浮窗字段增强方向

浮窗从快速计时器升级为快速任务记录器。

开始计时时字段：

- 任务名
- 一级分类：工作 / 学习 / 休息 / 生活
- 开始计时

结束计时后补充字段：

- 备注
- AI 赋能占比
- 完成状态
- 是否加入常用任务

2026-05-15 第一阶段补做：

- 右下角 Time Debt 浮窗空闲态新增 `一级分类` 选择。
- 固定选项：工作 / 学习 / 休息 / 生活 / 其他。
- 默认值：学习。
- 开始计时后 active timer 保留一级分类。
- 计时中卡片显示分类。
- 结束计时后写入 Time Debt log 的 `primaryCategory`。
- 最近任务点击时同时回填任务名和一级分类。

后续仍需单独开发结束计时补充框、AI 赋能占比和完成状态。

## 6. 快捷键策略

禁止使用：

- `Command+X`

原因：

- macOS 中 `Command+X` 是系统剪切快捷键。
- 输入框内使用会干扰文本编辑。
- 作为全局快捷键可能注册失败或造成误触。

推荐默认快捷键：

- Electron：`CommandOrControl+Alt+T`
- macOS 显示：`Cmd + Option + T`
- Windows / Linux 显示：`Ctrl + Alt + T`

fallback：

- `CommandOrControl+Shift+L`

实现约定：

- 主快捷键和 fallback 同时注册。
- fallback 不是功能入口的第二套实现，只是同一个 Time Debt 浮窗唤起动作的备用快捷键。

`CommandOrControl+Shift+Space` 不再作为默认主快捷键，因为它更容易与输入法、系统搜索或空间切换类快捷键产生冲突。

## 7. 后续开发顺序

1. 浮窗字段增强：任务名、一级分类、页面分类同步、新建任务名。
2. 结束计时补充框：备注、AI 赋能占比、完成状态。
3. Time Debt 首页仪表盘：今日分类占比、今日总时长、当前计时、计划 vs 实际。
4. Settings Foundation 接入：快捷键、时间单位、货币、安全线等配置统一收口。
