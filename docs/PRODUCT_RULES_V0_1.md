# PRODUCT_RULES_V0_1

## 产品定义

这不是普通笔记软件。  
它的目标是把每日复盘转译成成长树结构，并长期沉淀在本地桌面环境中。

## 节点准入规则

主图只允许以下节点类型进入：

- `mainline`
- `ability`
- `habit`
- `method`
- `issue`
- `cognition`
- `achievement`

以下内容不直接进入主图：

- 情绪流水
- 偶发事件
- 一次性吐槽
- 未确认的碎片记录

这些内容可以保留在复盘 markdown 中，但必须经过结构更新面板二次确认后，才进入成长树。

## 一级主线规则

本轮一级主线固定为 6 条：

1. 学业 / 认知成长
2. 身体 / 能量管理
3. 表达 / 输出能力
4. 关系 / 社会互动
5. 金钱 / 生存系统
6. 内在秩序 / 情绪与自我管理

本轮不开放自由新增主线。

## 节点状态规则

- 新建节点默认 `new`
- 节点再次被更新后转为 `growing`
- `evidence_count >= 3` 时转为 `stable`
- `dormant / review / restarted` 本轮仅保留字段，不自动触发

## 每日复盘转译流程

1. 用户先写自由文本复盘
2. 保存后生成 markdown 文件
3. 同步写入 `reviews` 表
4. 自动进入结构更新录入面板
5. 用户手动确认 1~3 条结构更新
6. 每条更新可选择：
   - 新建节点
   - 绑定已有节点
   - 是否记为证据
7. SQLite 更新 `nodes / edges / node_evidence`
8. 成长树刷新并展示新的时间信息

## 为什么本轮只做手动抽取

因为本轮核心不是“抽取得多聪明”，而是先把“复盘 -> 结构更新 -> 成长树”这条接口做成立。  
如果一开始就接自动抽取，会把精力消耗在提示词、召回、误判与修正体验上，反而会拖慢底座验证。

## 为什么主图只放稳定结构

成长树主图承担的是“长期结构回看”，不是“事件流水回放”。  
如果把所有复盘碎片直接铺到主图上，主视图会退化成噪声图，而不是成长结构图。
