# Life Vitality Tree 3D Concept

本文件定义 Life Vitality Tree / 人生生长树进入 3D 前的概念底座。本阶段只定义语义、数据驱动关系和视觉升级方向，不实现 Three.js / React Three Fiber，不导入模型，不改现有页面。

## 1. 核心定位

Life Vitality Tree 不是普通任务树，也不是固定成就树。它是一棵由人生数据驱动的生命体，用来表达一个人如何被底层支撑供给、如何沿主线生长、如何在阶段成果中结果、如何从失败与复盘中修复。

这棵树不是装饰性 3D 模型，而是人生数据的可视化载体。3D 只是表现层，真正的核心是数据、规则和快照如何描述一个人的长期生长状态。

3D 版本的目标不是先做一棵好看的树，而是先建立一套稳定的数据驱动骨架：

- 数据层描述今天、每周、每月和阶段发生了什么；
- 生长规则层把事件转换为树状态变化；
- 快照层保存某一天或某个阶段的树形状态；
- 渲染层只读取快照，不直接判断业务含义；
- 性能层按 Windows / Mac 能力选择不同渲染质量。

## 2. 树部位语义

| 树部位 | 人生含义 | 主要数据来源 | 3D 表现方向 |
| --- | --- | --- | --- |
| 根系 | 健康、睡眠、情绪稳定、复盘沉淀、财富安全线、关系支持、长期习惯 | 健康行动、复盘记录、财富安全线、关系事件 | 根须数量、深度、密度、亮度 |
| 主干 | 人生主线、长期方向、营养通道、长期持续性 | 主线、阶段目标、连续复盘、长期项目 | 高度、粗细、直立度、年轮 |
| 主枝 | 重要人生模块，例如项目、学习、健康、财富、关系、AI 协作 | 模块事件、项目进展、复盘分类 | 分叉方向、长度、粗细 |
| 小枝 | 主模块下的具体方向和阶段性路线 | 任务、习惯、子项目、阶段目标 | 分支数量、弯曲度、活跃度 |
| 叶子 | 每日行动、深度工作、健康行动、学习练习、轻量推进 | daily events、任务完成、复盘条目 | 叶片数量、颜色、透明度 |
| 花 | 阶段性中间成果，还未完全沉淀 | 小版本、跑通、初步完成、阶段验收 | 花苞或花朵数量 |
| 果实 | 可复用、可交付、可验证的阶段成果 | 发布、收益、获奖、交付、可复用方法 | 果实大小、成熟度、挂载枝干 |
| 裂痕 / 伤痕 | 时间负债、透支、失败、失控、长期未修复问题 | time debt、注意力泄漏、失败复盘 | 树皮裂痕、暗纹、局部断枝 |
| 年轮 | 年度 / 阶段复盘、长期成长轨迹、重大转折 | annual review、phase snapshot | 树干剖面或年轮视图 |
| 季节 | 当前生命状态，不等于现实季节装饰 | vitality pattern、成果、修复、透支状态 | 全局色调、落叶、光照、氛围 |

## 3. 影响树的人生模块

第一阶段保留以下现有模块作为数据驱动入口：

- 复盘：生成土壤、根系养分、修复信号和年轮材料。
- 时间负债：产生裂痕、枯叶、修复提醒和主干压力信号。
- 计划：提供未来枝干方向、阶段目标和待生长节点。
- 财富：影响财富枝、根系安全线和主干压力。
- 健康：影响根系供给、主干稳定度、季节状态和恢复速度。
- 项目：推动项目枝干、花和果实生长。
- 学习：推动学习枝、小枝和叶片持续生长。
- 关系：影响根系支持、外部风系统和季节调节。
- 成就：生成果实、年轮条目和阶段沉淀。

补充事件入口：

- 深度工作：主要推动项目枝、学习枝、AI 协作枝生长。
- 注意力泄漏：减少叶片鲜活度，增加枯叶、裂痕和生命力损耗。

## 4. 为什么不能一开始直接做写实模型

当前不能直接做写实模型，原因是语义和数据口径还未稳定：

- 如果先做整棵写实树模型，树的每个部位很难被数据独立控制。
- 写实模型成本高，改一次规则可能需要重新建模、绑定或拆资产。
- 现阶段真正未知的是数据如何驱动生长，而不是树皮材质如何渲染。
- Mac 无独显环境需要稳定降级，重模型会过早制造性能风险。
- 写实视觉容易掩盖规则缺陷，导致系统看起来完成但不可维护。

## 5. 为什么先做数据驱动骨架

先做数据驱动骨架可以保证后续升级路径清楚：

- 同一个 `TreeSnapshot` 可以驱动 2D、无 3D 模拟器、低模 3D 和写实 3D。
- 渲染层只负责表现快照，不负责解释业务数据。
- 后期写实模型可以替换树干、枝干、叶子、果实、裂痕等模块化资产。
- 规则和快照稳定后，再升级材质、模型、光照和动画，风险更小。

## 6. 写实化的正确方向

后期写实模型不是一整棵不可拆的死模型，而是一组模块化资产：

- trunk modules：不同年龄阶段和主线状态的树干模块；
- branch modules：不同模块、粗细、弯曲度的枝干模块；
- leaf modules：健康、枯萎、修复、季节状态的叶片模块；
- fruit modules：不同成果类型和成熟度的果实模块；
- scar modules：裂痕、断枝、树皮伤痕模块；
- root modules：根系深度、密度、土壤连接模块。

这些资产由 `TreeSnapshot` 中的状态字段选择和组合，而不是由渲染层临时推断人生含义。
