# PRODUCT_RULES_V0_3

## 当前状态同步

当前主线已切回：基础功能闭环优先。

P0.3 中关于图谱体验和 Obsidian 高保真方向的内容保留为历史规则与暂停支线参考，不作为下一轮默认主线或阻塞性验收标准。当前版本目标不是继续视觉还原，而是先保证产品状态稳定、基础闭环清楚、真实数据链路可接续。

## 本轮目标

P0.3 不再继续堆功能，而是把现有桌面工作区收口成更可用的图谱原型。

本轮闭环固定为：

`可点击工作区 -> 图谱聚焦 -> 提醒/周回看跳转 -> 右侧详情回看`

## 主画布表达规则

- 主画布继续使用成长树主线结构，但视觉表达改为更接近图谱的点状展开。
- 一级主线是稳定锚点，不再表现成大块树杈骨架。
- 普通节点以小型节点卡或点状节点为主，围绕所属主线展开。
- 连线用于表达关系与归属，不强调 rigid tree 层级。
- 搜索命中时只保留相关节点与其上级关系，避免整图噪音。

## Hover 浮层规则

- hover 触发以节点为中心，不跟随鼠标位置漂移。
- 默认显示在节点上方；若节点位置变化，浮层跟节点一起重新定位。
- hover 时：
  - 目标节点高亮
  - 一阶关联节点与连线保留强调
  - 其他节点与连线弱化
- 浮层最少显示：
  - 节点标题
  - 创建时间
  - 最近修改时间
- 时间格式统一为：`YYYY/MM/DD HH:mm`

## 提醒已处理规则

- 提醒状态仍只使用：
  - `open`
  - `done`
- `open` 提醒显示在 `待处理提醒`
- `done` 提醒显示在 `已处理提醒`
- 用户点击 `处理完成` 后，提醒不会直接消失，而是进入已处理分区。
- 已处理提醒仍保留：
  - 节点名
  - 提醒类型
  - 主线
  - 处理时间
  - 触发原因

## 周回看条目生成规则

统计窗口固定为最近 7 天。

除统计数字外，周回看还生成以下条目：

- 本周新增节点
  - `created_at` 在最近 7 天内
- 本周重复问题
  - `issue` 类型节点最近 7 天有新增证据，且符合重复问题规则
- 本周 dormant 节点
  - 当前状态为 `dormant`
  - 且进入 dormant 的时间落在最近一轮窗口内
- 本周 restarted 节点
  - 当前状态为 `restarted`
  - 且本周重新出现新证据

每条条目必须包含：

- 节点 id
- 节点标题
- 所属主线
- 当前状态
- 出现在这里的原因说明

## 跳转定位规则

以下入口都必须走同一套节点跳转动作：

- 提醒页 `查看节点`
- 周回看变化条目
- 复盘详情中的关联节点

统一动作顺序：

1. 切换到 `成长树` 视图
2. 选中目标节点
3. 视口平移到目标节点附近
4. 右侧切到节点详情
5. 节点进入聚焦态，其余非关联内容弱化

## 右侧详情规则

右侧详情区在 P0.3 内承担双模式：

- 节点详情
- 复盘详情

### 节点详情动作

- `这条我看过了`
  - 用于关闭回看类提醒
  - 若当前节点状态为 `review`，则回到 `stable`
- `处理这条提醒`
  - 用于关闭当前提醒
  - 不写入新证据
  - 不刷新 `last_active_at`

### 复盘详情动作

- 点击左侧最近复盘后，右侧切到复盘详情
- 复盘详情展示正文与本次复盘关联节点
- 点击关联节点后，触发统一节点跳转动作
